Critical Frontend Logic
1. Authentication Flow
Token Storage — httpOnly Cookies Only
Backend sets accessToken and refreshToken as httpOnly cookies. Frontend NEVER reads these directly. On page load, call GET /current-user (withCredentials: true). If 401 → auto-call POST /refresh-token → retry. Configure Axios interceptor for this.
Google OAuth Redirect Handling
When landing on /?login=success, call /current-user to hydrate user state. The backend already set cookies — frontend just needs to pick up the user object.
Registration — 3-Step OTP Flow
Step 1: POST /send-otp. Store email in state. Step 2: POST /verify-otp with 6-digit code (auto-submit on 6th digit). OTP has 10 min expiry — show countdown timer. Resend button with 60s cooldown. Step 3: POST /register with all fields including avatar multipart.
2. Video Player Logic
Watch Progress Tracking (Critical)
Save progress every 5 seconds via setInterval. POST /watch-progress/save with {videoId, currentTime, duration}. On mount, GET /watch-progress/:videoId and seek player to savedTime if > 10s. At 90% completion, mark isCompleted=true (triggers interest score: +10 "complete" vs +5 "watch").
Interest Engine Triggers
Call updateUserInterests (via backend) for: watch (+5), complete (+10), like (+8), comment (+6), subscribe (+12). This happens automatically via the backend when you call like/comment/subscribe APIs — but ensure you call these APIs, not just update UI optimistically.
Watch History
Automatically tracked server-side when video is fetched. No separate call needed. GET /users/watch-history returns populated video objects.
Slug-based vs ID-based Navigation
Videos have both _id and slug. Use slug in URLs (/watch/my-video-slug) for SEO. Fetch by videoId which accepts both. The slug is auto-generated from title via slugify on upload.
3. Subscribe Button
Optimistic Update + Anti-Self-Subscribe
Toggle immediately in UI, then POST to API. Backend rejects if channelId === own userId (400 error) — hide subscribe button on own channel. Show subscriber count from API response. isSubscribed state comes from channel profile endpoint.
4. Shorts Feed
Vertical Snap Scroll + Intersection Observer
Use IntersectionObserver to detect which short is in view. Autoplay video when fully visible (threshold: 0.9). Pause others. Each short is 100vh. Use scroll-snap-type: y mandatory on container. Preload next short.
5. Recommendation Feed
Personalized vs Guest Feed
For authenticated users: GET /recommendations/videos. For guests: GET /videos/get-all-videos?sortBy=views&sortType=desc. The backend recommendation engine uses categoryScores and channelScores — no frontend logic needed, just call the right endpoint.
6. Like System
Optimistic Updates on Likes
Toggle like button state immediately. POST /likes/toggle/video/:videoId. Update likesCount from response. If unauthenticated, redirect to /login. Likes work on: videos, comments, tweets. Check current like state from video/comment data (isLiked field).
7. Notifications Bell
Polling for Unread Count
Poll GET /notifications/unread-count every 60 seconds when user is authenticated. Show badge count on bell icon. On click: fetch all notifications and PATCH /notifications/mark-all-read. Types shown: new_video (show thumbnail), new_tweet (show text preview).
8. Search
Debounced Search with URL Sync
Debounce input 300ms. Update URL query param ?q= for shareable results. GET /search/query?q=. Show results grouped: Videos, Channels (users), Shorts. Navigate to /search?q=term on enter or suggestion click.
9. Upload Flow
Video Upload with Progress
POST /videos/upload-video as FormData. Show upload progress via onUploadProgress (Axios). Fields: title (required), description (required), category, isShort toggle, videoFile, thumbnail. After success: redirect to /watch/:videoId. Notifications auto-sent to subscribers by backend.
Component Architecture
Layout Components
Navbar
Logo left. Search bar center (with debounce). Right: Notification bell (with unread badge), Upload button, Avatar/Login button. Hamburger for mobile sidebar toggle.
Sidebar
Collapsible. Routes: Home, Shorts, Subscriptions, Watch Later, History, Liked Videos, Playlists. Auth-gated routes show login prompt. Creator section: Studio, Community.
CategoryChips
Horizontal scrollable filter row below navbar. Fetched from /categories API. "All" chip first. Selected chip highlighted red. On click, filter home feed.
Video Components
VideoCard
Thumbnail (16:9), title (2 lines max), channel avatar+name, views, timestamp, duration badge. Hover: show progress bar if watched. Bookmark icon for watch later.
VideoPlayer
React Player wrapper. Custom controls: play/pause, volume, fullscreen, quality selector, seek bar. Progress tracking interval. Resume from saved position.
ShortCard
Full-height (100vh). Vertical video. Like button, share button. Channel name + subscribe overlay. Uses autoplay when in viewport via IntersectionObserver.
CommentSection
Comment count header. Input box (auth required). Paginated comments with like toggle. Owner can edit/delete own comments. Sorted by newest.
Shared Components
SubscribeButton
Red "Subscribe" / Gray "Subscribed" with checkmark. Handles own-channel hide. Shows subscriber count. Optimistic toggle.
LikeButton
Thumbs up icon. Active = red fill. Shows count. Works on videos, tweets, comments. Redirects to login if guest.
OTPInput
6 individual digit boxes. Auto-focus next on input. Auto-submit on 6th digit. Resend with 60s countdown timer. Clear on error.
AvatarUpload
Circle preview. Drag-drop or click to select. Shows preview before upload. Used in register and settings.
InfiniteScroll
IntersectionObserver on sentinel div. Triggers next page fetch. Works for videos, notifications, comments, playlists.
NotificationItem
New video: thumbnail + channel avatar + "uploaded a video". New tweet: text preview. Unread = highlighted bg. Timestamp relative.
State Management (Zustand Stores)
authStore
user, isAuthenticated, login(), logout(), fetchCurrentUser(). Hydrated on app mount via /current-user.
playerStore
currentVideo, isPlaying, volume, watchProgress. Syncs with backend every 5s.
notificationStore
notifications[], unreadCount. Polled every 60s.
uiStore
sidebarOpen, theme (dark/light), searchQuery.
Cursor AI Implementation Guide
Use this as your prompt guide in Cursor. Start with project scaffold, then build page by page. Always reference the API Base URL from .env: VITE_API_BASE_URL=http://localhost:8000/api/v1
Project Setup Prompt
Create a React + Vite project with: React Router v6, TanStack Query, Zustand, Tailwind CSS, Axios with withCredentials:true base config, React Player. Dark/light theme via CSS variables. Tailwind config with YouTube-inspired colors: primary=#FF0000, dark-bg=#0F0F0F, surface=#212121.
Axios Config Prompt
Create axios instance with baseURL from env, withCredentials:true. Add request interceptor. Add response interceptor: on 401 → call /refresh-token → retry original request. On refresh fail → logout user and redirect /login.
Auth Store Prompt
Zustand auth store: user state, isAuthenticated. fetchCurrentUser calls GET /users/current-user (optional auth endpoint). Login/logout functions. Persist to sessionStorage. Add ProtectedRoute component that redirects to /login if not authenticated.
